Introduction

1. Creating a Todo

To create a todo, click in the text field at the top right and write your todo (For mobile: click on the menu symbol in the top left, if you do this the todo creating menu will appear). Once this is done, you can set the quantity setting to determine whether your todo should have a quantity (e.g. if food is missing: 5x flour, etc.) If the check mark is set, this means with quantity; the default setting for the quantity is 1.

2. Changing the suffix of the quantity number

To change the suffix, click on the cogwheel in the upper right corner. Once you have done this, a settings menu will appear. The suffixes available are a dot, an x, the word "Times" and no suffix at all. To make a setting now, hover over the various settings on the right. when you do this, you will see small grey arrows. if you click on these, the green arrow will move and the new setting will be active.

3. The "Complete" function and the deletion of todos

The "Complete" function

To mark a todo as finished, click on the bookmark to the far left of your todo. When this is done, you will notice that the text is greyed out and a line has been drawn through the text. The todo is now marked as finished and can be reactivated at any time by simply clicking on the bookmark again. Note: you can only use the bookmark every three seconds.

Deleting todos

To delete todos, click on the checkbox to the far right of the todo you want to delete. When you do this, the todo is completely deleted from the system and you cannot restore it.

4. Difference between the local and public version of the todo list

The local todo list

The local todo list stores the user data on the device (therefore local) and retrieves it from there. The advantage is that you do not need a permanent internet connection because the data is stored on the device; you only need a connection to load the page. Everyone therefore has their own local todo list, as no other device can access it.

The public todo list

The public todolist variant, on the other hand, stores the data in databases located on the server, which means that you need an internet connection to call up the todos or to delete or add them (so you actually need an internet connection all the time). The advantage of this is that the todo list is shared with everyone and you can see when others delete or add a todo.
